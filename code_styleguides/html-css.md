# HTML/CSS Code Style Guide

This document outlines the HTML and CSS coding conventions for this project.

## HTML
- Indentation: Use 2 spaces.
- Tag names: `lowercase`.
- Attributes: `lowercase`.
- Semantic HTML: Use appropriate semantic tags (e.g., `<header>`, `<nav>`, `<main>`, `<article>`, `<footer>`).
- Accessibility: Include `alt` attributes for images, use proper ARIA roles when necessary.

## CSS
- Indentation: Use 2 spaces.
- Naming: `kebab-case` for classes and IDs.
- Organization: Group related properties. Use comments for sections.
- Specificity: Keep selectors as low specificity as possible.
- Units: Use `rem` or `em` for font sizes; `px` for borders and small fixed values.
- Responsive design: Use media queries for different screen sizes (mobile-first approach).
