# Python Code Style Guide

This document outlines the Python coding conventions for this project, adhering to PEP 8.

## PEP 8 Adherence
- Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) for all code.

## Formatting
- Indentation: Use 4 spaces.
- Line length: Max 79 characters.
- Quotes: Prefer single quotes for strings, unless double quotes allow escaping. (e.g., "I'm a string").

## Naming Conventions
- Modules: `snake_case`
- Packages: `snake_case`
- Classes: `PascalCase`
- Functions: `snake_case`
- Variables: `snake_case`
- Constants: `SCREAMING_SNAKE_CASE`

## Best Practices
- Docstrings for all public modules, functions, classes, and methods.
- Type hints for functions and variables.
- List comprehensions over explicit loops where appropriate.
